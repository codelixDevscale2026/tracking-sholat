# Issue Writing Guide

When creating an issue, please follow this template to ensure all necessary information is provided for a quick and effective resolution.

## Problem (Overview)
Provide a clear and concise description of the problem or the context of the issue. 
- What are you trying to achieve? 
- Why is this an issue?

## Current Behavior
Describe what is currently happening.
- Be as specific as possible.
- If it's a bug, include steps to reproduce the issue.
- Include error logs, screenshots, or any visual aids if applicable.

## Expected Behavior
Describe what you expected to happen instead of the current behavior.
- How should the system behave when functioning correctly?

## Relevant Files
List any files, directories, or modules that are related to this issue.
- This helps narrow down where the problem might be originating from or what files need to be modified.

## Task
Outline the specific actionable steps required to resolve the issue.
- Break down the work into smaller tasks if necessary.
- You can use checkboxes (e.g., `- [ ] Task step`) to track progress.
